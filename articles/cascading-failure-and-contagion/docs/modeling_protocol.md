# Cascading Failure and Contagion Modeling Protocol

Cascade modeling treats propagation, exposure, thresholds, network structure, recovery, and containment as explicit analytical objects.

## Core questions

- What counts as failure, infection, adoption, overload, default, or functional loss?
- What spreads through the system?
- Which network or dependency layer carries propagation?
- Are links directed, weighted, multiplex, delayed, or capacity constrained?
- What thresholds determine state change?
- Which nodes are seeded by the initial shock?
- Does failure redistribute load, transmit risk, or change behavior?
- How fast does recovery occur relative to propagation?
- Which containment strategies reduce cascade size?
- How sensitive are results to thresholds, topology, and missing dependencies?

## Companion workflow

The companion workflows compare baseline threshold, lower-threshold, higher-connectivity, larger-shock, high-threshold containment, sparse-network, dense-low-threshold, and small containment scenarios.
