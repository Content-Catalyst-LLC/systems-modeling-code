# Validation Protocol

Validation is purpose-specific. For this synthetic companion model, validation means checking that outputs remain interpretable and that scenario behavior matches expected structural logic.

## Checks

- Scripts run from a clean checkout.
- Scenario inputs are documented.
- Outputs are reproducible.
- State remains inside expected ranges.
- Longer delays produce greater overshoot risk.
- Weaker balancing feedback produces weaker correction.
- Higher growth increases stress on the system.
- Stronger threshold correction constrains high-state behavior.
- Sensitivity diagnostics identify influential assumptions.

## Professional extensions

- Historical comparison
- Expert review
- Alternative model structures
- Stakeholder review
- Calibration to empirical data
- Stress testing under extreme conditions
