# Validation Checklist

A systems model is validated for a purpose, not validated in general.

## Basic checks

- [ ] Code runs from a clean checkout.
- [ ] Inputs are documented.
- [ ] Outputs are reproducible.
- [ ] Units are consistent.
- [ ] Extreme conditions behave plausibly.
- [ ] Model structure matches the stated purpose.
- [ ] Parameters are documented as measured, estimated, assumed, or scenario-based.
- [ ] Sensitivity analysis has been run.
- [ ] Outputs are interpreted with uncertainty.
- [ ] Known limitations are documented.

## Higher-standard checks

- [ ] Historical behavior reproduction where appropriate.
- [ ] Out-of-sample validation where appropriate.
- [ ] Expert review.
- [ ] Stakeholder review.
- [ ] Alternative model comparison.
- [ ] Cross-platform reproducibility.
- [ ] Stress testing under extreme conditions.
