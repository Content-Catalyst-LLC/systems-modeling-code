# Model Boundary

A systems model should begin with an explicit boundary.

## Included in this companion model

- Synthetic infrastructure and public-system dependency network
- Weighted directed edges
- Shock propagation
- Redundancy
- Recovery capacity
- Coupled stock-flow recurrence
- Monte Carlo parameter uncertainty
- Scenario comparison
- Validation diagnostics

## Excluded from this companion model

- Real infrastructure data
- Real empirical calibration
- Actor-level behavioral adaptation
- Institutional politics
- Geographic exposure
- Equity distribution
- Maintenance logistics
- Cascading repair constraints
- Real-time sensor data

## Boundary critique questions

- What costs or harms are outside the model?
- Whose experience is not represented?
- What feedback loops are simplified?
- Which assumptions would change if the model were used in a real decision context?
