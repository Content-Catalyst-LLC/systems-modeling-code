# Interface Contracts

Interfaces are the seams of a hybrid model. They should be documented like formal model components.

## Required fields

- Source module
- Target module
- Exchanged variable
- Units
- Exchange frequency
- Transformation rule
- Valid range
- Missing-value behavior
- Uncertainty representation

## Companion interfaces

The `data/interface_inventory.csv` file documents the synthetic interfaces used in this folder.

## Professional caution

Many hybrid models fail because the interfaces are underdefined, not because the individual modules are weak.
