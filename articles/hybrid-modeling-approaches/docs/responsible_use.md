# Responsible Use

Hybrid models can appear comprehensive because they combine multiple methods. That apparent comprehensiveness can create false authority.

Do not use these synthetic workflows for real decisions without:

- Module-level validation
- Interface validation
- Uncertainty propagation
- Stakeholder review
- Documentation of exclusions
- Sensitivity analysis
- Data governance
- Reproducible run instructions

A hybrid model should clarify how system layers interact. It should not hide assumptions inside coupling logic.
