# Assumptions and Limitations

## Assumptions

- All data are synthetic.
- The structural baseline is intentionally incomplete.
- The learned residual captures a simplified nonlinear correction.
- Validation metrics are illustrative.
- Governance tables are conceptual.
- No real AI deployment, dataset, system, or organization is represented.

## Limitations

- The Python workflow uses a simple gradient-descent linear residual learner with engineered features.
- The R workflow uses base R linear modeling rather than external machine-learning packages.
- These examples demonstrate modeling patterns, not production AI.
- Applied use requires privacy review, domain validation, subgroup testing, drift monitoring, security review, model documentation, and accountability mechanisms.
