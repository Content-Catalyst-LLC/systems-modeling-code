# Model Assumptions and Boundary Judgment Protocol

Assumption and boundary work should begin before model construction and continue through interpretation.

## Core questions

- What is the model for?
- What system is being represented?
- What is inside the model boundary?
- What is outside the boundary?
- Which assumptions are structural, causal, parameter, data, behavioral, scenario, scale, measurement, or normative?
- Which assumptions are uncertain, sensitive, or consequential?
- Which exclusions could change the interpretation?
- Which stakeholders should review the boundary?
- What conclusions are valid only under a narrow boundary?
- What uses of the model should be prohibited?

## Companion workflow

The companion workflows score assumption risk, compare boundary scenarios, summarize categories, and export validation checks.
