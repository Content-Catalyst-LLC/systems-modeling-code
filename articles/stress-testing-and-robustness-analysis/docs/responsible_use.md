# Responsible Use

Stress testing and robustness analysis can reduce false reassurance, but they can also create false confidence if scenario design is narrow or biased.

Do not use these synthetic workflows for real decisions without:

- Domain data
- Justified stress scenarios
- Explicit failure thresholds
- Model validation
- Structural review
- Sensitivity and robustness checks
- Distributional and equity review
- Clear communication of what was and was not tested

Surviving a stress test does not prove full resilience.
