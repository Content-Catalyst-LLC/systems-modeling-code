# Early Warning Signal Modeling Protocol

Early-warning modeling treats weakening recovery, rising fluctuation, spatial reorganization, network fragility, and monitoring design as explicit analytical objects.

## Core questions

- What system and failure mode are being monitored?
- What mechanism could produce collapse or regime shift?
- Which variables reflect system function, recovery, stress, or capacity?
- Which warning indicators are theoretically plausible?
- Are the data long enough, frequent enough, and clean enough?
- Do warning indicators survive alternative windows and preprocessing choices?
- Does a structural model explain why the indicator should rise?
- What interventions are triggered by warning signals?
- How are false positives and false negatives communicated?

## Companion workflow

The companion workflows compare baseline destabilization, moderate destabilization, high noise, low noise, short-window, long-window, rapid-destabilization, and slow-destabilization scenarios.
