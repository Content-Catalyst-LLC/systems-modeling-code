# Assumptions and Limitations

## Assumptions

- All data are synthetic.
- The physical system is represented as a single hidden state.
- Observations are noisy proxies for the true state.
- Twin synchronization uses a transparent update rule.
- Anomaly detection uses a simple residual threshold.
- Intervention logic is illustrative.
- Governance tables are conceptual.

## Limitations

- This is not a calibrated digital twin.
- It does not represent real sensors, operational systems, assets, geospatial dynamics, cybersecurity conditions, or organizational decision processes.
- It does not implement a full Kalman filter, particle filter, physics model, optimization engine, or production data pipeline.
- Applied use requires engineering validation, data governance, cybersecurity review, privacy assessment, model documentation, drift monitoring, human oversight, and operational testing.
