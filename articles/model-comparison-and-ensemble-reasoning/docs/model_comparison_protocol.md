# Model Comparison Protocol

Model comparison should begin with purpose.

## Core questions

- Is the comparison predictive, structural, scenario-based, or decision-oriented?
- Are model outputs comparable?
- Are units and time horizons aligned?
- Which models are benchmarks?
- Which models are structural alternatives?
- What validation data are held out?
- Are ensemble members independent?
- Are weights justified?
- What would disagreement imply?

## Companion workflow

The companion code compares exponential, logistic, managed-logistic, equal-weight ensemble, and performance-weighted ensemble models using synthetic data.
