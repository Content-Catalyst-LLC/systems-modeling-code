# Causal Loop Translation Notes

Causal loop diagrams are useful for identifying feedback hypotheses. They are not enough for simulation unless translated into stock-flow structure.

## Translation workflow

1. Identify the behavior over time.
2. Identify candidate reinforcing and balancing loops.
3. Identify the stocks that accumulate system memory.
4. Identify inflows and outflows.
5. Add delays where response depends on earlier conditions.
6. Define nonlinear constraints, thresholds, and saturation.
7. Write equations and check units.
8. Simulate baseline behavior.
9. Compare scenarios.
10. Validate and document assumptions.

## Caution

A causal loop diagram can be persuasive even when the underlying model is not valid. Professional system dynamics requires stock-flow translation, dimensional consistency, testing, and clear interpretation.
