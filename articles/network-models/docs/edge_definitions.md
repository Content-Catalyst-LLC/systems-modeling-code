# Edge Definitions

Edges in this companion model represent generalized dependency or interaction pathways.

## Edge fields

- `source`: first endpoint
- `target`: second endpoint
- `weight`: relative connection strength
- `capacity`: illustrative functional capacity
- `edge_type`: local bridge or cross-layer

## Interpretation

Edges are undirected in the core workflows. They can transmit contagion or dependency effects in either direction.

## Professional extension

In applied work, define whether edges represent physical connection, contact, obligation, influence, capacity, shared exposure, contractual dependency, or information flow. These meanings should not be mixed without documentation.
