# Feedback Modeling Protocol

Feedback modeling translates recursive causal structure into executable systems models.

## Core questions

- What dynamic behavior needs explanation?
- Which variables change through time?
- Which causal links form closed loops?
- Which loops are reinforcing, balancing, delayed, nonlinear, adaptive, networked, or stock-flow based?
- Which stocks accumulate pressure?
- Which delays alter behavior?
- Which loops dominate at different times?
- Which interventions activate counter-feedback?

## Companion workflow

The companion workflows simulate reinforcing, balancing, logistic, delayed balancing, stock-flow, and policy-resistance feedback structures.
