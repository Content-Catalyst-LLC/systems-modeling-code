# Responsible Use

Feedback-loop models can reveal recursive harms, unintended consequences, and hidden dependencies. They can also create false authority if speculative loops are presented as evidence.

Do not use these synthetic workflows for real decisions without:

- Domain evidence
- Boundary review
- Structural validation
- Parameter justification
- Sensitivity and uncertainty analysis
- Stakeholder review when public consequences are involved
- Clear communication of what is and is not modeled

Feedback analysis should improve accountability. It should not make responsibility disappear into "the system."
