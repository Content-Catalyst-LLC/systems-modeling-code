# Responsible Systems Modeling Checklist

## Before Building
- Define the modeling purpose.
- Define the system boundary.
- Identify stakeholders and affected groups.
- List key stocks, flows, agents, nodes, or events.
- State known uncertainties.

## During Modeling
- Document assumptions.
- Preserve parameter values.
- Identify feedback loops.
- Check units and time scales.
- Run sensitivity tests.
- Compare scenarios.

## Before Interpretation
- Separate exploration from prediction.
- Report uncertainty and limitations.
- Explain boundary exclusions.
- Avoid overstating model precision.
- Connect conclusions to model purpose.
