# Clarification-Distortion Review Protocol

A systems model should be reviewed for what it clarifies, what it distorts, and what uses it can responsibly support.

## Core questions

- What is the model for?
- What does the model clarify?
- What does the model distort or hide?
- Which assumptions drive conclusions?
- Which boundaries are narrow or contested?
- Which proxies are being used?
- Are uncertainty and sensitivity visible?
- Does validation support the intended use?
- Could communication convert conditional outputs into false certainty?
- Who remains accountable for the decision?

## Companion workflow

The companion workflows score clarification value, distortion risk, and net interpretive value for several synthetic model-use cases.
