# Climate Resilience Scenario Modeling Protocol

This case models resilience as dynamic service performance under climate stress.

## Core steps

1. Define the system function or service being protected.
2. Define climate stress trajectories.
3. Define exposure and sensitivity.
4. Define adaptive capacity and recovery.
5. Define degradation and threshold logic.
6. Define adaptation investment pathways.
7. Simulate service performance over time.
8. Track threshold exposure, degradation, transformation, and resilience diagnostics.
9. Compare scenarios.
10. Communicate uncertainty, valid use, and limitations.

## Core logic

Resilience is not only bouncing back. It is the ability to maintain, recover, adapt, or transform function under stress.
