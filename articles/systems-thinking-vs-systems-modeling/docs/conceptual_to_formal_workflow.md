# Conceptual to Formal Workflow

This companion folder demonstrates how systems thinking can be translated into systems modeling.

## Translation steps

1. Define the behavior of interest.
2. Map conceptual relationships.
3. Identify stocks, flows, feedback loops, delays, and leverage points.
4. Decide which conceptual terms become variables.
5. Define equations, rules, or algorithms.
6. Run scenarios.
7. Compare conceptual expectations with formal model behavior.
8. Revise the map and the model.

## Key caution

Not every concept should become a variable. Translation requires judgment, and every formal model simplifies the system it represents.
