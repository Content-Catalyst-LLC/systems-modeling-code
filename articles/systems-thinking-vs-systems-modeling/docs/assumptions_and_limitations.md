# Assumptions and Limitations

The workflows in this folder are synthetic. They demonstrate professional modeling structure.

## Assumptions

- Demand, capacity, backlog, trust, rework, and learning can be represented as state variables.
- Structural redesign and intervention pressure affect the system differently.
- Conceptual systems reasoning can be compared with modeled system behavior.
- Scenario comparison can reveal where conceptual intuition and formal dynamics diverge.

## Limitations

- No empirical calibration is included.
- Variables are illustrative.
- Stakeholder values are simplified.
- Institutional power and politics are represented only indirectly.
- Results should not be used as real-world evidence.
